## PRODUCTION NOTES FOR RECORDING

### Screen Recording Setup
- Record on a real Android device (or high-quality emulator at 1080p)
- Use screen mirroring to OBS or similar — keep phone in portrait
- Record at 60fps if possible — card animations look much better
- Keep the Antigravity trace panel visible for at least 20 continuous seconds during Act 2

### Narration Tips
- Record narration separately, sync in post
- Speak slowly and clearly — judges may not be native English speakers
- Pause 1-2 seconds after each trace panel update so judges can read it
- Keep personal opener genuine — emotion matters more than polish

### B-roll to Insert (optional but powerful)
- 2-3 seconds of a real child tapping a tablet (royalty free stock or your own)
- Pakistan street/home background for the opening
- Animated star / Sitara character for transitions

### Captions
- Add English captions throughout (accessibility + judges watching on mute)
- For Urdu words shown on screen, add English subtitle below

## 🎯 Problem Statement

Pakistan has approximately **350,000 children diagnosed with autism**, the majority of whom are non-verbal. Existing AAC (Augmentative and Alternative Communication) tools are:

- Designed for English-speaking Western contexts
- Expensive and inaccessible in rural Pakistan  
- Not adaptive to individual child behaviour in real time
- Missing Urdu, Roman Urdu, and culturally relevant content (Eid, chai, roti, dadi)
- Reliant on camera-based emotion detection (privacy invasive, unreliable in low light)

**Sitara solves all five problems** — using Google Antigravity to power real-time agentic adaptation based entirely on tap behaviour (no camera required).

## 🤖 Google Antigravity Usage

This is the heart of our submission. Antigravity is not a wrapper here — it is the **game engine**.

### Agent 1: Therapy Director
**Trigger:** Every 30 seconds during an active game session

**Role:** Observes session metrics, infers emotional/cognitive state, adapts the game

**Frustration Signals Detected:**
| Signal | Threshold | Response |
|--------|-----------|----------|
| Consecutive failures | ≥ 3 | Switch to easier/preferred category |
| Tap speed | > 3 taps/sec | Reduce difficulty + trigger reward |
| Inactivity | > 30 seconds | Send break prompt |
| Session length + declining success | 15+ min, < 60% | Suggest break |

**Engagement Signals Detected:**
| Signal | Threshold | Response |
|--------|-----------|----------|
| Success rate | > 80% for last 10 | Increase difficulty |
| Category repeat | Voluntary return | Log preference signal |
| Consistent tap rhythm | 1-2 taps/card | Log focused engagement |

**Tools Called:**
- `get_session_state` — fetch live metrics
- `switch_category` — move to preferred/easier category
- `adjust_difficulty` — change card count (2-6) and size
- `trigger_reward` — star/confetti animation + Urdu audio praise
- `generate_quest` — request personalised story from Story Weaver
- `send_break_prompt` — gentle stretch/water/hug break suggestion
- `log_insight` — record observation for Progress Guardian

**Reasoning Protocol (visible in trace panel):**
```
OBSERVE → What signals am I seeing?
INFER   → What is the child's emotional/cognitive state?
DECIDE  → What single adaptation will help most right now?
ACT     → Call the appropriate tool
LOG     → Plain language explanation for parent report
```

### Agent 2: Story Weaver
**Trigger:** On `generate_quest` tool call from Therapy Director, or child session start

**Role:** Generates 3-5 sentence personalised mini-adventures that give card recognition a narrative purpose

**Cultural Grounding:**
- Eid, chai, roti, samosa, mango (aam), cricket
- Local animals: billi (cat), kutta (dog), gaaye (cow)
- Family terms: ammi, abu, dadi, dada, bhai
- Urdu praise: Shabash!, Wah wah!, Bohat acha!, Shero!, Kamaal!

**Output Schema:**
```json
{
  "quest_title": "Sitara's Lost Kitten",
  "story_text": "Mubarak ho! Sitara's billi has run away...",
  "target_category": "animals",
  "character": "Sitara",
  "urdu_hook": "Mubarak ho, Zara!",
  "difficulty": "easy"
}
```

### Agent 3: Progress Guardian
**Trigger:** Weekly, or on parent request from dashboard

**Role:** Synthesises multi-session data into warm, actionable parent reports

**Report Sections:**
1. Warm personalised greeting (Assalamu Alaikum + child name)
2. This Week's Wins (3 achievements with evidence and numbers)
3. What Your Child Loves (preference patterns)
4. Gentle Observations (struggles framed constructively — never as failure)
5. Suggested Home Activity (1 offline action for parents)
6. Next Week Preview (what Sitara will introduce)
7. Encouragement Close (warm, personal)

**Tone Rules:**
- ✅ "Zara recognised 12 new symbols — up from 7 last week!"
- ✅ "She lights up with animal cards. Try pointing on your walk!"
- ❌ Never: "Zara failed emotion cards 14 times"
- ❌ Never: Clinical language like "deficit", "impairment", "disorder"

## 🎮 Game Design

### Symbol Cards
- **5 Categories:** Animals, Food, Family, Emotions, Daily Routines
- **19 starter cards** with Urdu / Roman Urdu / English labels
- **3 Difficulty Levels:** Level 1 (concrete nouns), Level 2 (emotions), Level 3 (actions)
- **Offline audio:** Pre-recorded Urdu TTS for every card

### Frustration Detection (Privacy-Safe)
We deliberately avoided camera-based emotion detection. Instead, we use **tap behaviour as a frustration proxy**:

- **Tap speed** (taps/second): rapid tapping = frustration signal
- **Consecutive failures**: 3+ = high frustration threshold
- **Inactivity duration**: 30+ seconds = disengagement signal
- **Success rate trend**: declining over last 10 interactions = fatigue

This approach is:
- Privacy-preserving (no camera, no biometrics)
- Works in all lighting conditions
- Culturally appropriate
- Computationally lightweight (works offline)

### Adaptation Window
The Therapy Director makes at most **one adaptation per 60-second window** to avoid overwhelming the child. Positive reinforcement is always preferred over difficulty reduction alone.

## 🌍 Pakistan-Specific Design Decisions

| Decision | Rationale |
|---------|-----------|
| Urdu + Roman Urdu + English labels | Pakistan's linguistic reality — parents read all three |
| Offline-first with local fallback | Unreliable connectivity in many Pakistani cities |
| Cultural symbol content | Mango, chai, roti, Eid, dadi — familiar & engaging |
| Android-only (initial) | 95%+ of Pakistani smartphone users are on Android |
| No subscription paywall in MVP | Accessibility over monetisation for hackathon |
| Urdu audio TTS | Many parents are Urdu-dominant; English TTS is alienating |

## 🚧 Assumptions & Limitations

### Assumptions
- Children have 1-on-1 adult supervision during sessions
- Primary caregiver can read at least basic Urdu or Roman Urdu
- Android device with ≥ 2GB RAM (required for Flutter + audio)
- Internet connectivity available for at least initial session (agent calls); offline fallback for subsequent use

### Current Limitations (Hackathon MVP)
- Symbol library is limited to 19 cards across 5 categories (full version: 200+ cards)
- Antigravity API calls use mock responses in demo mode (real keys required for live)
- No therapist account — single parent + child profile only
- No multi-child household support
- English-only app UI chrome (Urdu content on cards; full Urdu UI is post-hackathon)
- No data sync across devices (local SQLite only)

### Post-Hackathon Roadmap
1. **Week 6-8:** Expand to 200+ symbols, therapist portal, cloud sync
2. **Month 3:** Urdu full UI, parent community features
3. **Month 6:** Multi-language (Punjabi, Sindhi, Pashto)
4. **Month 12:** Partner with autism NGOs in Pakistan for pilot deployment

## 💰 Cost & Scalability

### Per-session cost estimate (Gemini 2.0 Flash, May 2026)

| Item | Value |
|------|-------|
| Input tokens per evaluation call | ~500 tokens |
| Output tokens per evaluation call | ~200 tokens |
| Evaluation calls per 10-min session | 20 (every 30 s) |
| Approx. cost per session | ~$0.002 |

Pricing basis: Gemini 2.0 Flash ~$0.075/1M input tokens, ~$0.30/1M output tokens.

### Scaling

| Scale | Sessions / day | Est. daily cost | Infrastructure note |
|-------|---------------|-----------------|---------------------|
| MVP (hackathon) | 50 | $0.10 | Cloud Run free tier; SQLite session DB |
| 10× | 500 | $1.00 | Cloud Run auto-scales; SQLite sufficient |
| 100× | 5 000 | $10.00 | Switch `DatabaseSessionService` to Cloud SQL; batch Progress Guardian reports |

### Latency

Gemini 2.0 Flash p50 response: ~800 ms. Flutter's 30-second evaluation window absorbs this comfortably with no visible latency to the child.

### Baseline comparison evidence

The 🤖 AI / 📏 Rules toggle in the game AppBar lets judges run side-by-side sessions. The Parent Dashboard Mode Comparison card shows average success rate per mode — demonstrating measurable engagement improvement from the agentic system.
